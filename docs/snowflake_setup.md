# Snowflake Setup

This project uses Snowflake as the primary warehouse. Airflow runs locally in Docker and connects to Snowflake with the credentials in `.env`.

## Free-Trial-Friendly Objects

Use a small warehouse for portfolio runs:

```sql
USE ROLE ACCOUNTADMIN;

CREATE WAREHOUSE IF NOT EXISTS NASDAQ_ELT_WH
    WAREHOUSE_SIZE = XSMALL
    AUTO_SUSPEND = 60
    AUTO_RESUME = TRUE
    INITIALLY_SUSPENDED = TRUE;

CREATE DATABASE IF NOT EXISTS NASDAQ_MARKET_DATA;
CREATE ROLE IF NOT EXISTS NASDAQ_ELT_ROLE;
```

## Least-Privilege Grants

```sql
GRANT USAGE ON WAREHOUSE NASDAQ_ELT_WH TO ROLE NASDAQ_ELT_ROLE;
GRANT USAGE ON DATABASE NASDAQ_MARKET_DATA TO ROLE NASDAQ_ELT_ROLE;
GRANT CREATE SCHEMA ON DATABASE NASDAQ_MARKET_DATA TO ROLE NASDAQ_ELT_ROLE;
GRANT ROLE NASDAQ_ELT_ROLE TO USER YOUR_USER_NAME;
```

After the pipeline creates schemas, grant ownership or schema-level privileges if you prefer to pre-create them:

```sql
USE DATABASE NASDAQ_MARKET_DATA;

CREATE SCHEMA IF NOT EXISTS UTIL;
CREATE SCHEMA IF NOT EXISTS BRONZE;
CREATE SCHEMA IF NOT EXISTS SILVER;
CREATE SCHEMA IF NOT EXISTS GOLD;

GRANT ALL PRIVILEGES ON ALL SCHEMAS IN DATABASE NASDAQ_MARKET_DATA TO ROLE NASDAQ_ELT_ROLE;
GRANT ALL PRIVILEGES ON FUTURE TABLES IN DATABASE NASDAQ_MARKET_DATA TO ROLE NASDAQ_ELT_ROLE;
GRANT ALL PRIVILEGES ON FUTURE VIEWS IN DATABASE NASDAQ_MARKET_DATA TO ROLE NASDAQ_ELT_ROLE;
```

## `.env` Values

```bash
SNOWFLAKE_ACCOUNT=your-org-your-account
SNOWFLAKE_USER=your_user
SNOWFLAKE_PASSWORD=your_password
SNOWFLAKE_ROLE=NASDAQ_ELT_ROLE
SNOWFLAKE_WAREHOUSE=NASDAQ_ELT_WH
SNOWFLAKE_DATABASE=NASDAQ_MARKET_DATA
```

Do not include `.snowflakecomputing.com` in `SNOWFLAKE_ACCOUNT`.

## Cost Controls

- Keep `NASDAQ_SYMBOL_LIMIT=3` for the first run.
- Use `ALPHA_VANTAGE_OUTPUTSIZE=compact`.
- Keep the warehouse at `XSMALL`.
- Confirm `AUTO_SUSPEND = 60`.
- Suspend the warehouse manually after demos if needed:

```sql
ALTER WAREHOUSE NASDAQ_ELT_WH SUSPEND;
```
